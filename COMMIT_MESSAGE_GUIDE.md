# Commit Message Guide

Use clear, action-oriented commit messages so history is easy to understand.

## Recommended format

`type(scope): short summary`

Examples:

- `feat(profile): add endpoint to update display name`
- `fix(auth): handle expired JWT gracefully`
- `refactor(service): simplify user lookup flow`
- `test(controller): add integration test for missing user`
- `docs(readme): clarify local setup instructions`

## Good commit message checklist

- Start with a meaningful type: feat, fix, refactor, test, docs, chore
- Keep the summary under 72 characters when possible
- Use imperative voice (add, fix, update)
- Explain why in the commit body when the change is not obvious
